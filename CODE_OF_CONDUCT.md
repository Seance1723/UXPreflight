# Code of Conduct

## Our Pledge

We want UXPreflight to be a welcoming, respectful, and useful open-source project. We pledge to make participation in this project a positive experience for contributors, maintainers, users, and community members.

## Expected Behavior

- Be respectful and considerate.
- Assume good intent, especially when giving or receiving feedback.
- Keep discussions focused on the project and its users.
- Give clear, constructive feedback.
- Welcome beginners and help them find a useful path to contribute.
- Respect different backgrounds, experiences, and skill levels.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language, imagery, or attention.
- Repeated disruption of project discussions.
- Publishing private information without permission.
- Dismissing people instead of addressing ideas.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or limit participation when behavior violates this Code of Conduct. Enforcement should be fair, proportionate, and focused on keeping the project healthy.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers privately. Until a dedicated contact is configured, use the security contact listed in `SECURITY.md` and include "Code of Conduct" in the subject.

Reports should include enough context for maintainers to understand what happened. Maintainers will handle reports with care and limit sharing to what is necessary.

## Scope

This Code of Conduct applies in project spaces, including GitHub issues, pull requests, discussions, documentation, community chat, and any public or private interaction related to UXPreflight.
