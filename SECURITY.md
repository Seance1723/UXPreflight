# Security Policy

## Supported Versions

For now, `v0.1.x` alpha is supported for security reports.

## Reporting a Vulnerability

Please do not open public GitHub issues for sensitive security problems.

Report vulnerabilities privately to:

```text
security@example.com
```

Until a dedicated security email is configured, maintainers can replace this address with the correct contact.

## Local-First Security Principle

UXPreflight is designed to work locally and should avoid sending project files, design rules, or source code to external services by default.

## Sensitive Data

Do not include secrets, API keys, private credentials, customer data, or proprietary source code inside rule files, issue reports, screenshots, or reproduction repositories unless you are certain it is safe to share.

## Responsible Disclosure

When reporting a vulnerability, please provide:

- Description
- Steps to reproduce
- Affected version
- Impact
- Suggested fix if available

Maintainers will review reports as quickly as possible and coordinate disclosure when a fix is available.
